# VPSL Structure Claim Template

Use this template when archiving a candidate physical structure for K3 and
later milestones.

## Structure

Name the physical structure or prior.

## System

Name the physical system or benchmark.

## Regime

State the validated regime and the evidence that the regime gate passed.

## Controls

List baseline, fair controls, and any controls excluded as degenerate.

## Primary Metric

Define the metric used for the main performance claim.

## Mechanism

Define the diagnostic used to test whether the proposed structure caused the
effect.

## Transfer Horizon

State the transfer setting, stress horizon, or out-of-regime test.

## Verdict

Use the VPSL verdict taxonomy, for example:

- `REJECTED`
- `NO_EFFECT`
- `PERFORMANCE_ONLY`
- `MECHANISM_CONFIRMED`
- `BOUNDED_POSITIVE`
- `TRANSFER_CONFIRMED`
- `FULL_TRANSFER_CONFIRMED`

## Non-Claims

List boundaries explicitly: systems not tested, structures not compared,
controls not available, and generalizations not claimed.
