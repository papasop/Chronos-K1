# Claim Boundary Map

| Layer | Authority | Supports | Does Not Support | Next Gate |
| --- | --- | --- | --- | --- |
| K2 | physics validation | FPU-beta symplectic transfer through H=240 | all physical priors work; universal physics AI | wrong-Omega hardening and broader-system transfer |
| K3 | active physics investigation | active topology / GP regime search evidence | K3 certified; topology prior validation | K3.2D transport gate and baseline regime validation |
| L-VPSL | grounded language rendering | bounded no-LLM grounded language through L5 | open-domain language; LLM-level fluency; real robot deployment | real sensor/action trace replay |
| Y30 | contextualize only | cognitive substrate and K-family context bridge | Y30 is physics evidence; K-family verdicts are changed by Y30 | Y30-L1 appearance vs object boundary |
| Y20 | audit / required gate only | objection / response / required-gate self-audit | physics claims are resolved by debate; scientific realism is refuted | Y20-L1 objection taxonomy coverage |
| VPSL | promotion authorization | decides whether a structure may be promoted from evidence | bypassing required regime, control, mechanism, or transfer gates | next required gate for the structure family |
| ClaimRecord | audit ledger | records supports, does_not_support, evidence_level, next_gate, and claim_boundary | proving claims by narration alone | replay and external review |

Chronos uses these boundaries to keep positive evidence from becoming an
overclaim. A layer may contextualize, audit, or render a claim only within the
evidence it actually has.
